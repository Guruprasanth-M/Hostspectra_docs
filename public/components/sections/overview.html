Overview

Why HostSpectra Exists

I run a simple Linux server — just hosting my site, nothing enterprise. I wanted to know if someone was brute-forcing my SSH, if something weird was running, if an IP was scanning my ports. Basic stuff. Should be simple.

First I tried Splunk — built for companies with dedicated security teams. The setup alone requires reading documentation for days. Then I tried Wazuh — same story. A separate indexer, a separate manager, agents to install, Elasticsearch underneath, 4 GB RAM minimum just to start.

Every tool I found was an enterprise product built for enterprise teams. Heavy. Complex. My server runs my actual site — I can't dedicate all of that just to watch it.

But Docker is already on my server. Redis runs in Docker. My app runs in Docker. Why can't security monitoring just be another docker compose up?

So I built it. Everything I needed — logs, processes, network connections, ML anomaly scoring, auto IP blocking, real-time dashboard — all in containers, one command, on the same cheap server that runs my site without it even noticing.

HostSpectra — because every server deserves a guard that never sleeps.

What It Does

🔍 Real-Time Monitoring

Collects CPU, memory, disk, network, processes and ports from host every 5 seconds via /proc and psutil

🤖 ML Anomaly Detection

3-method ensemble scoring: Isolation Forest + Z-score + EMA deviation for intelligent threat detection

📋 Policy Engine

YAML-based alert rules with severity levels (critical, high, medium, low) and configurable cooldown periods

⚡ Automated Response

IP blocking via iptables, process termination with safety guardrails, and webhook notifications to Slack/Discord

📊 Live Dashboard

Glassmorphism UI with WebSocket-powered real-time updates, metrics visualization and alert management

🐳 Zero Host Agents

Everything runs in Docker; reads host data via bind-mounted /proc filesystem — no host installation required
