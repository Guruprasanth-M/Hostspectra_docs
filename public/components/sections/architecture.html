System Architecture

Data Flow Pipeline

HostSpectra processes security data through a seven-stage streaming pipeline via Redis Streams:

1. Collect

3 parallel collectors gather logs, network, processes from /proc

→

2. Aggregate

Feature builder creates 12-number vectors every 5s

→

3. Score

ML ensemble: Isolation Forest + Z-score + EMA

→

4. Evaluate

Policy engine checks YAML rules and severity

→

5. Act

IP blocking, process termination with guardrails

→

6. Notify

HMAC-signed webhooks to Slack/Discord

→

7. Display

Dashboard via WebSocket real-time updates

Container Services (10 Containers)

All services run on private Docker network (hostspectra-net). Only port 8080 exposed to host.

📝 Log Collector

Tails auth.log, syslog, kern.log — classifies events via regex

🌐 Network Collector

Parses /proc/net/tcp, detects port scans and new connections

⚙️ Process Collector

Snapshots via psutil, flags CPU/memory spikes and suspicious parents

🧠 ML Engine

Isolation Forest anomaly scoring with automatic retraining

📋 Policy Engine

YAML rule evaluation with severity levels and cooldowns

⚔️ Action Engine

Automated IP blocking and process termination with safety checks

📡 Webhook Service

Alert delivery with retry queue and HMAC signing

🔗 API Server

FastAPI REST API + WebSocket streaming

📊 Dashboard

Nginx + Vanilla JavaScript glassmorphism SPA

💾 Redis

Event bus, stream storage, consumer groups, data persistence
